# 🚀 GitHub Actions Auto-Deploy to RunPod

Your GitHub repository now has **automatic Docker build and RunPod deployment**!

## How It Works

```
You push code to GitHub
    ↓
GitHub Actions builds Docker image
    ↓
Image pushed to ghcr.io (GitHub Container Registry)
    ↓
RunPod template automatically updated
    ↓
Your endpoint instantly has the latest code!
```

---

## ⚙️ Setup (One-Time Only)

### Step 1: Create RunPod Template Manually

First, create the template ONE TIME on RunPod:

1. **Go to:** https://www.runpod.io/console/templates
2. **Click:** + New Template
3. **Fill in:**
   - Name: `gemma-4-31b-runpod`
   - Docker Image: `ghcr.io/corecomai/gemma-4-31b-runpod:latest`
   - Container Disk: `50 GB`
   - Volume: `50 GB`

4. **Add Environment Variables:**
   ```
   MODEL_NAME=google/gemma-4-31b-it
   QUANTIZATION=int4
   TENSOR_PARALLEL_SIZE=1
   MAX_MODEL_LEN=32768
   GPU_MEMORY_UTILIZATION=0.90
   MAX_NUM_SEQS=64
   VLLM_WORKER_MULTIPROC_METHOD=spawn
   HF_TOKEN={{ RUNPOD_SECRET_HF_TOKEN }}
   ```

5. **Click:** Save Template

### Step 2: Get Template ID

After saving, your template gets an ID (looks like: `abc123xyz`).

**Copy it** - you'll need it next.

### Step 3: Add Repository Variables

1. **Go to:** https://github.com/Corecomai/gemma-4-31b-runpod/settings/variables/actions

2. **Click:** + New repository variable

3. **Add Variable:**
   - Name: `RUNPOD_TEMPLATE_ID`
   - Value: `your_template_id_here`

4. **Click:** Add variable

### Step 4: Verify Secrets

Check that these secrets exist:

**Go to:** https://github.com/Corecomai/gemma-4-31b-runpod/settings/secrets/actions

You should have:
- ✅ `RUNPOD_API_KEY` (your RunPod API key)
- ✅ `GITHUB_TOKEN` (auto-generated by GitHub)

If `RUNPOD_API_KEY` is missing:

1. **Click:** + New repository secret
2. **Name:** `RUNPOD_API_KEY`
3. **Value:** `your_runpod_api_key_here`
4. **Click:** Add secret

## 🔍 Troubleshooting

### Workflow Failed

**Check logs:**
1. Go to: https://github.com/Corecomai/gemma-4-31b-runpod/actions
2. Click the failed workflow
3. Look for error messages

**Common Issues:**

| Error | Fix |
|-------|-----|
| `RUNPOD_API_KEY not found` | Add secret in settings |
| `RUNPOD_TEMPLATE_ID not found` | Add variable in settings |
| `Docker build failed` | Check Dockerfile syntax |
| `Template update failed` | Verify template ID is correct |

### Endpoint Not Updating

**Check:**
1. Workflow completed successfully (✅ in Actions tab)
2. Template ID is correct
3. RunPod API key is correct

## 📚 What Each File Does

```
.github/workflows/docker-build-deploy.yml
├── Build job
│   ├── Builds Docker image from Dockerfile
│   ├── Pushes to ghcr.io/corecomai/gemma-4-31b-runpod
│   └── Outputs digest (image version hash)
│
└── Deploy job
    ├── Gets digest from Build job
    ├── Uses RunPod API to update template
    ├── Pushes new image to your template
    └── Your endpoint auto-updates
```
